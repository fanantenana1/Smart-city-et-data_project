  curl -H "Authorization: Bearer <token>" \
    http://localhost:8000/api/export/json
  
  # CSV spécifique
  curl -H "Authorization: Bearer <token>" \
    http://localhost:8000/api/export/csv/poubelles \
    -o poubelles.csv
  
  # SQLite
  curl -H "Authorization: Bearer <token>" \
    http://localhost:8000/api/export/sqlite \
    -o rapport.db
  
  # Tous formats
  curl -H "Authorization: Bearer <token>" \
    http://localhost:8000/api/export/all

🔌 ENDPOINTS API DISPONIBLES
───────────────────────────────────────────────────────────────────────────────

GET /api/export/json
├─ Retourne: Toutes données JSON
├─ Auth: JWT Required
└─ Response: {"poubelles":[...], "historiques":[...], ...}

GET /api/export/csv/{table}
├─ Paramètres: table = poubelles|historiques|predictions|users
├─ Auth: JWT Required
└─ Response: Fichier CSV téléchargeable

GET /api/export/sqlite
├─ Retourne: Base SQLite avec 4 tables
├─ Auth: JWT Required
└─ Response: Fichier .db téléchargeable

GET /api/export/sql-dump
├─ Retourne: SQL dump compatible MySQL/PostgreSQL
├─ Auth: JWT Required
└─ Response: Fichier .sql téléchargeable

GET /api/export/excel
├─ Retourne: Classeur avec 4 sheets
├─ Auth: JWT Required
├─ Dépendances: pandas, openpyxl
└─ Response: Fichier .xlsx téléchargeable

GET /api/export/all
├─ Lance tous les exports
├─ Retourne: Infos fichiers créés
├─ Auth: JWT Required
└─ Response: {"fichiers": {...}, "status": "success"}

GET /api/export/info
├─ Retourne: Infos exports disponibles
├─ Statistiques MongoDB
└─ Taille estimée

🐛 TROUBLESHOOTING
───────────────────────────────────────────────────────────────────────────────

Problème: "Excel non disponible - pandas not found"
Solution: pip install pandas openpyxl

Problème: "401 Unauthorized"
Solution: Vérifier token JWT valide, pas expiré (24h)

Problème: "404 poubelles not found"
Solution: Exécuter python generate_massive_data.py d'abord

Problème: "Fichiers exports/ vides"
Solution: Vérifier permissions: chmod 755 backend/exports

Problème: "Timeout export"
Solution: Limiter période ?period=week ou augmenter timeout
